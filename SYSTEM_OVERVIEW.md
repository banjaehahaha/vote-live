# 발표·투표 시스템 사용 가이드 (검수용)

코드가 아닌 **시스템이 어떻게 동작하는지** 검수할 때 참고하는 문서입니다.

---

## 1. 시스템 개요

- **역할**: 1회 발표 이벤트에서 **실시간 투표**를 받고, **투표 결과 순위(1~4위)**에 따라 **발표 화면(장면)**을 순서대로 보여주는 시스템입니다.
- **역할 구분**  
  - **발표자/운영**: 대형 화면에 `/presentation`을 띄우고 단계를 진행합니다.  
  - **관객**: 휴대폰 등에서 QR/링크로 `/v`에 접속해 4개 선택지 중 하나를 투표합니다.
- **세션(sid)**: 한 이벤트 = 한 세션 ID. 같은 sid로 발표 화면·투표·스크린이 묶입니다. 현재 기본 sid는 **test-0311**입니다.

---

## 2. 접속 가능 주소

배포 시 **https://vote.banjaeha.com** (또는 배포된 도메인) 기준입니다.

| 주소 | 용도 | 비고 |
|------|------|------|
| `/` | 홈 | 투표/스크린 링크 제공 (sid=test-0311) |
| `/mobile` | 단축 URL | 접속 시 `/v?sid=test-0311`로 리다이렉트 (QR용) |
| `/mobile?sid=원하는세션` | 단축 URL + 세션 | `/v?sid=원하는세션`으로 이동 |
| `/presentation` | **발표자용 메인** | sid 없으면 test-0311 사용. 전체 발표 단계 진행 |
| `/presentation?sid=세션ID` | 발표자용 + 세션 지정 | 해당 세션으로 발표 진행 |
| `/v` | **관객 투표** | **sid 필수**. 예: `/v?sid=test-0311` |
| `/v?sid=세션ID` | 관객 투표 (세션 지정) | 해당 세션에 투표 |
| `/screen?sid=세션ID` | 스크린 집계 화면 | 투표 수 실시간 표시 (sid 없으면 test-0311) |
| `/r/ITEM`, `/r/IMAGE`, `/r/DATA`, `/r/NEAR` | 결과 카드 | 투표 후 이동. 쿼리에 `?sid=세션ID` 사용 |

- **같은 이벤트**로 쓰려면 발표·투표·스크린 모두 **같은 sid**를 사용해야 합니다.

---

## 3. 발표 플로우 (presentation 단계)

`/presentation`에서 단계는 아래 순서로 진행됩니다. (뒤로 가기는 없고, 일부 단계에서만 버튼/단축키로 제어합니다.)

| 순서 | 단계 (영문) | 설명 |
|------|-------------|------|
| 1 | **setup** | 네트워크 확인, 세션(sid) 표시. "시작" 버튼 클릭 시 다음 단계로 |
| 2 | **intro** | 인트로 화면. 클릭 또는 Enter → 다음 |
| 3 | **vote** | 투표 화면. QR·투표 URL·4개 선택지·**투표 시작/종료/결과 확정/다시 시작** 등 |
| 4 | **locked-result** | 확정된 1~4위 순위 표시. "발표 시작" 클릭 → 다음 |
| 5 | **present-existing-works** | 1위→2위→3위→4위 순서로 각 작업(choice)별 장면을 보여줌. 화살표/N 키로 진행 |
| 6 | **residency-plan** | 레지던시 계획 장면 5개. 화살표로 진행, 마지막에서 다음 시 end |
| 7 | **end** | 종료 화면 |

- **vote** 단계에서만 관객 투표가 가능합니다.  
  “투표 시작”을 눌러야 관객이 `/v`에서 투표할 수 있고, “투표 종료” 후 “결과 확정”을 눌러 순위를 고정해야 **locked-result**와 **present-existing-works**에 반영됩니다.

---

## 4. 발표 화면(presentation) 단축키

**vote 단계가 아닐 때** (intro, locked-result, present-existing-works, residency-plan 등) 아래 키가 동작합니다.  
입력창/셀렉트에 포커스가 있으면 동작하지 않습니다.

| 키 | 동작 |
|----|------|
| **Enter** | intro에서 다음 단계로 진행 |
| **← (왼쪽 화살표)** | 이전 장면 / 이전 레지던시 장면 |
| **→ (오른쪽 화살표)** | 다음 장면 / 다음 레지던시 장면 (마지막에서 누르면 end로) |
| **N** | present-existing-works에서 **다음 작업(다음 순위)**으로 이동 |
| **P** | present-existing-works에서 **이전 작업(이전 순위)**으로 / residency-plan에서 **이전 장면** |

- **present-existing-works**: 1위 장면들을 보다가 **N**을 누르면 2위 → 3위 → 4위로 넘어가고, 4위 다음에는 **residency-plan**으로 자동 전환됩니다.
- **residency-plan**: **←** 또는 **P**로 이전 장면, **→**로 다음 장면. 마지막 장면에서 **→**를 누르면 **end**로 갑니다.

---

## 5. vote 단계에서의 조작 (버튼)

| 버튼 | 표시 시점 | 동작 |
|------|-----------|------|
| **투표 시작** | 투표 전(idle) | 1분 카운트다운 시작, 관객이 `/v`에서 투표 가능 |
| **투표 종료** | 투표 중(running) | 투표 마감, 더 이상 접수 안 함 |
| **결과 확정** | 투표 중(running) / 투표 종료 후(closed) | 현재 집계로 1~4위 확정 후 **locked-result**로 이동 |
| **다시 시작** | 투표 중(running) / 투표 종료 후(closed) | 해당 세션 투표·집계 초기화 후 처음부터 “투표 시작” 상태로 |
| **직접 설정** | 항상 | 수동으로 순위/시간 설정 (Plan B) |
| **🔊/🔇** | 항상 | 효과음 on/off |

- 실수로 “결과 확정”을 눌렀다면, **“다시 시작”**으로 초기화한 뒤 다시 “투표 시작”부터 진행하면 됩니다.

---

## 6. 스크린·투표 페이지

- **스크린** (`/screen?sid=test-0311`): 해당 sid의 4개 선택지별 득표 수를 실시간으로 표시. 1초 간격으로 갱신됩니다.
- **투표** (`/v?sid=test-0311`): 4개 버튼(ITEM, IMAGE, DATA, NEAR) 중 하나 선택 시 해당 세션에 투표되고, 결과 카드 페이지로 이동합니다.

---

## 7. 검수 시 확인하면 좋은 것

1. **접속**  
   - `/presentation` 접속 → setup → 시작 → intro → vote 순서로 진행되는지  
   - `/v?sid=test-0311` 접속 시 4개 선택지가 보이고, 투표 시 “전송중…” 후 결과 카드로 넘어가는지  

2. **투표 흐름**  
   - vote 단계에서 “투표 시작” 클릭 → 1분 카운트다운 시작  
   - 다른 기기/탭에서 `/v?sid=test-0311`로 투표 → 발표 화면·스크린에 숫자 증가  
   - “투표 종료” → “결과 확정” → locked-result에 1~4위 표시  

3. **단축키**  
   - present-existing-works에서 **← / →** 로 장면 이동, **N**으로 다음 작업(다음 순위) 이동, **P**로 이전 작업 이동  
   - residency-plan에서 **← / →** 또는 **P**로 장면 이동, 마지막에서 **→** 시 end로 이동  

4. **다시 시작**  
   - vote 단계에서 “투표 시작” 후 “다시 시작” 클릭 → 카운트·집계 초기화되고 다시 “투표 시작”만 보이는지  

5. **세션 분리**  
   - `/v?sid=test-0311`와 `/v?sid=other`는 서로 다른 세션으로, 스크린/발표에서도 sid가 같을 때만 해당 세션 투표가 보이는지

## 8. 요약

| 구분 | 내용 |
|------|------|
| **발표 진행** | `/presentation` (기본 sid: test-0311) |
| **관객 투표** | `/v?sid=test-0311` 또는 QR(/mobile) |
| **실시간 집계** | `/screen?sid=test-0311` |
| **장면 이동** | ← / → (이전/다음), N(다음 작업), P(이전 작업/이전 장면) |
| **투표 제어** | 투표 시작 → 1분 카운트 → 투표 종료 → 결과 확정 (또는 다시 시작) |

문의나 오동작 발견 시 **어느 단계(예: vote, present-existing-works)**에서 무엇을 했을 때인지 알려주면 원인 파악에 도움이 됩니다.
